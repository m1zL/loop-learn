# M7: 学習モード拡張 — 精緻的質問 & インターリービング学習

> 状態: ⬜ 未着手
> 対象期間: 目安 1〜2週間
> 依存: M3完了 (復習エンジンが必要)

---

## 概要

科学的学習法を深化させる2つの学習モードを追加する。
精緻的質問モードは答えを確認した後に「なぜ？」プロンプトを提示し、深い理解を促す。
インターリービング学習モードは複数デッキをシャッフル出題し、知識の応用力を鍛える。

---

## 含む機能・タスク

### 精緻的質問・自己説明モード

- [ ] 復習セッション中の「答えを確認する」後に精緻化プロンプトをオプション表示
  - [ ] 「なぜそうなるのか？」「具体例を1つ挙げると？」など問いかけ文のセット
  - [ ] カードの `cardType` と `tags` に基づいて適切な問いを選択するロジック
- [ ] セッション設定画面に「精緻的質問モードをON」トグルを追加
- [ ] ユーザー設定に `elaborativeQuestionEnabled: boolean` を追加

### インターリービング学習モード

- [ ] 「インターリービング復習」ページ `/review/interleave`
  - [ ] 対象デッキを複数選択できる UI (チェックボックス)
  - [ ] 選択デッキの本日の復習予定カードをシャッフルして出題
- [ ] `getTodayReviewCards` に複数 `deckId` 配列フィルタを追加
- [ ] AppNav / ダッシュボードに「混合復習」リンクを追加

### テスト

- [ ] 精緻化プロンプト選択ロジックのユニットテスト
- [ ] インターリービングのカードシャッフルロジックのユニットテスト

### 品質チェック

- [ ] `npm run lint` がパスする
- [ ] `npm run typecheck` がパスする
- [ ] `npm test` がパスする

## 受け入れ条件

- [ ] 精緻的質問モード ON 時、答え確認後に「なぜ？」プロンプトが表示される

## 実装メモ

- 精緻化プロンプトはハードコードのセット (例: 10〜20問) で十分。AI生成は不要
- インターリービングのシャッフルは `Fisher-Yates` アルゴリズムで実装
- `elaborativeQuestionEnabled` の永続化は `localStorage` でMVP十分。DBカラムは不要
