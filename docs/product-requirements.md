# プロダクト要求定義書 (Product Requirements Document)

## プロダクト概要

### 名称
**loop-learn** - アクティブリコール × 間隔反復で学びを定着させる学習ツール

### プロダクトコンセプト
- **科学的学習法の実践**: アクティブリコール・間隔反復・精緻的質問など、認知科学に基づく学習法をプロダクトの中心に据える
- **エンジニア特化の入力体験**: 書籍・ドキュメント・コードスニペット・図解など、webエンジニアが日常で触れる学習素材に対応した入力フォーマットを提供
- **モチベーション維持の仕組み**: 学習記録の可視化・連続学習ストリーク・復習リマインダーにより、継続的な学習をサポート

### プロダクトビジョン
webエンジニアが書籍や技術ドキュメントから得た知識を、確実に長期記憶へ転換できる学習ツールを提供する。
単なるフラッシュカードアプリに留まらず、アクティブリコールや間隔反復といった科学的学習法を自然に実践できる体験を設計する。
スマートフォンのすき間時間にも学習できる快適なUI/UXにより、忙しいエンジニアでも継続できる学習習慣の形成を支援する。

### 目的
- インプットした技術知識の記憶定着率を向上させる
- 科学的学習法(アクティブリコール・間隔反復等)をエンジニアが無理なく実践できる環境を提供する
- 学習継続のモチベーションを高め、長期的な学習習慣を形成させる
- webエンジニアのスキルアップに必要な各技術領域の理解を深める

## 成功指標(KPI)

### プライマリーKPI
- **月次アクティブユーザー(MAU)**: 500人(リリース6ヶ月後)
- **7日間継続率**: 40%以上(初回利用から7日後も使用継続しているユーザーの割合)
- **平均セッション数**: 週4回以上/ユーザー(定着した学習習慣の指標)

### セカンダリーKPI
- **カード作成数**: 平均50枚以上/アクティブユーザー/月
- **復習完了率**: スケジュールされた復習の70%以上を完了
- **ユーザー満足度(NPS)**: 30以上(利用開始30日後・以降は月次でアプリ内アンケートを配信)

## 機能要件

### コア機能(MVP)

#### 1. 学習カード作成

**ユーザーストーリー**:
webエンジニアとして、書籍や技術ドキュメントから学んだ知識をアウトプットするために、複数の入力フォーマットに対応した学習カードを作成したい

**受け入れ条件**:
- [ ] 以下のカードタイプが作成できる
  - Q&A形式(問題文と答え)
  - 穴埋め形式(テキスト中のキーワードを隠す)
  - コードスニペット形式(コードブロック付きの問題)
  - 自由記述形式(白紙勉強法・ファインマンテクニック対応)
- [ ] Markdown記法でカード本文を入力できる
- [ ] Mermaid記法でシーケンス図・フローチャート・ER図などをカード内に描画できる
- [ ] カードにタグ・デッキ(学習領域)を紐付けられる
- [ ] カード作成時間が1枚あたり1分以内で完了できる操作性

**優先度**: P0(必須)

---

#### 2. 間隔反復アルゴリズムによる復習スケジューリング

**ユーザーストーリー**:
学習者として、記憶の定着効率を最大化するために、「忘れかけたタイミング」で自動的に復習スケジュールを提示してほしい

**受け入れ条件**:
- [ ] SM-2アルゴリズム(SuperMemoベース)または同等の間隔反復アルゴリズムを実装
- [ ] 各カードに対して自己評価(1〜4段階: 全然わからない / うっすら / わかった / 完璧)を入力できる
- [ ] 自己評価に基づいて次回復習日が自動計算・更新される
- [ ] 本日の復習予定カード一覧をトップ画面に表示する
- [ ] 復習予定日を過ぎたカードを「期限切れ」として優先表示する

**優先度**: P0(必須)

---

#### 3. 復習セッション

**ユーザーストーリー**:
学習者として、隙間時間に効率的に復習するために、スムーズに次のカードへ進める復習セッションを体験したい

**受け入れ条件**:
- [ ] セッション開始から最初のカードが表示されるまで2秒以内
- [ ] カードの問題面を表示し、ユーザーが回答を考えた後に答え面をフリップできる
- [ ] 答え確認後に自己評価(4段階)を選択できる
- [ ] セッション中の進捗(残り枚数・完了枚数)を常に表示する
- [ ] セッションを任意のタイミングで中断・再開できる
- [ ] セッション終了時に「本日の成果サマリー」(復習枚数・平均評価・連続学習日数)を表示する

**優先度**: P0(必須)

---

#### 4. デッキ管理

**ユーザーストーリー**:
エンジニアとして、技術領域別に学習コンテンツを整理するために、デッキ(カードの集合)を作成・管理したい

**受け入れ条件**:
- [ ] デッキの作成・編集・削除ができる
- [ ] デッキに名前・説明・アイコン(絵文字またはプリセットアイコン)を設定できる
- [ ] デッキ一覧画面でカード枚数・本日の復習予定枚数を確認できる
- [ ] デッキ単位での復習セッションを開始できる

**優先度**: P0(必須)

---

#### 5. 学習進捗ダッシュボード

**ユーザーストーリー**:
学習者として、自分の学習状況を把握してモチベーションを維持するために、学習記録をビジュアルで確認したい

**受け入れ条件**:
- [ ] 過去30日間の学習カレンダー(ヒートマップ形式)を表示する
- [ ] 連続学習日数(ストリーク)を目立つ位置に表示する
- [ ] 総学習カード数・習熟度別カード分布(未学習/学習中/習得済み)を表示する
- [ ] 週次・月次の復習完了率をグラフで表示する

**優先度**: P0(必須)

---

#### 6. AIによる問題自動生成

**ユーザーストーリー**:
エンジニアとして、学習領域のカードを素早く揃えるために、テキストを入力するだけでQ&Aカードを自動生成したい

**受け入れ条件**:
- [ ] テキストブロック(書籍の1節・ドキュメントの一部など)を入力すると、5〜10問のQ&Aカードを自動生成できる
- [ ] 生成されたカードをユーザーが編集・削除・追加してから保存できる
- [ ] 技術ドメイン(フロントエンド / バックエンド / インフラ / アルゴリズム / OS / ネットワーク等)を指定して問題の難易度・観点を調整できる
- [ ] 生成にかかる時間が30秒以内

**優先度**: P0(必須)

### 将来的な機能(Post-MVP)

#### 7. 精緻的質問・自己説明モード

学習者が「なぜ？」「どうして？」を自問するプロンプトをカード表示時に提示し、深い理解を促す。

**優先度**: P1(重要)

#### 8. インターリービング学習モード

複数のデッキから横断的にカードをシャッフルして出題し、知識の応用力を高める学習モード。

**優先度**: P1(重要)

---

#### 9. コーネルメソッドノート連携

**優先度**: P1(重要)

#### 10. 学習リマインダー通知

設定した時刻にプッシュ通知/メール通知で復習リマインダーを送る機能。

**優先度**: P1(重要)

**優先度**: P2(できれば)

## 非機能要件

### パフォーマンス
- ページ初期読み込み時間: 2秒以内(3G回線想定)
- カードめくり(フリップ)アニメーション: 16ms以内(60fps)
- AI問題自動生成: 30秒以内にレスポンス

### ユーザビリティ
- 初めて使うユーザーが10分以内に最初のデッキを作成し復習セッションを完了できる
- モバイル(iOS / Android Chrome)でのタップ操作に完全対応
- 画面幅375px〜1440pxのレスポンシブデザイン
- フォントサイズ・コントラスト比はWCAG 2.1 AA準拠

### 信頼性
- 学習データ(カード・復習履歴)の損失ゼロ: サーバーサイドに永続化
- サービス可用性: 99.5%以上/月
- 復習スケジュールの計算誤差: 0日(正確なスケジュール管理)

### セキュリティ
- ユーザー認証: メール/パスワード認証 + ソーシャルログイン(Google)
- 通信の暗号化: HTTPS必須
- 他ユーザーのプライベートデッキへのアクセス禁止

### スケーラビリティ
- 1ユーザーあたり最大5,000枚のカードに対応
- 同時接続ユーザー1,000人での安定動作

## スコープ外

以下は明示的にスコープ外とする:

- ネイティブアプリ(iOS / Android)の開発 ※PWA対応でモバイル利用に対応
- 動画・音声コンテンツの学習サポート
- 他ユーザーとのリアルタイムコラボレーション機能
- 企業向けチーム管理・一括アカウント管理機能
- オフライン完全動作(ネット接続を前提とする)
